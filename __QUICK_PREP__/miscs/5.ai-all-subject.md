## Core Java Fundamentals

### 9. What are the main features of Java?
- Platform-independent (Write Once, Run Anywhere)
- Object-oriented programming language
- Secure
- Robust and reliable
- Multithreaded
- Compiled and Interpreted

### 10. What is the difference between JDK, JRE, and JVM?
- **JDK (Java Development Kit)**: Contains JRE + tools to compile Java into class files (javac)
- **JRE (Java Runtime Environment)**: Contains libraries to run Java code
- **JVM (Java Virtual Machine)**: Loads, verifies, and executes class files

### 11. Explain JVM internal working
- Class Loading → Bytecode Verification → Execution Engine → Garbage Collection
- Reference: https://medium.com/platform-engineer/understanding-jvm-architecture-22c0ddf09722

### 12. What is the superclass of every class in Java?
- Every class implicitly extends `Object` class
- It is the root of class hierarchy

### 13. What is the difference between public static void main(String args[])?
- **public**: Method can be called from anywhere (JVM needs to invoke it)
- **static**: No need to create object; JVM can call it directly
- **void**: Method doesn't return anything
- **main**: Standard entry point for Java program
- **String[] args**: Command-line arguments

### 14. Can we have multiple main methods in a Java project?
- **In a single class**: Not allowed (only one main method per class)
- **In project level**: Allowed in different classes (IDE will prompt which class to run)
- **Overloading**: Can overload main method by changing parameters
- **Overriding**: Cannot override main method (static method cannot be overridden)

### 15. Is the main method compulsory in Java?
- Not always required if using frameworks like Spring Boot
- Compulsory in standalone Java applications
- Reference: https://www.geeksforgeeks.org/main-method-compulsory-java/

### 16. What is the difference between program arguments and VM arguments?
- **Program Arguments**: Passed to main method via `String[] args`
- **VM Arguments**: Passed to JVM and affect JVM behavior (e.g., -Xmx1024m for max heap memory)

### 17. What are the different data types in Java?
- **Primitive**: byte, short, int, long, float, double, boolean, char
- **Non-primitive (Reference)**: String, Arrays, Classes, Interfaces

### 18. What is the difference between String literals and new String()?
- **String literal ("abc")**: Created in String pool; multiple references point to same object if value is same
- **new String("abc")**: Always creates new object in heap memory
- **Memory locations**: String pool (for literals) vs Heap (for new String)

### 19. What is the String pool?
- Special memory area in heap to store String literals
- Purpose: Avoid creating duplicate String objects; improves memory efficiency
- Managed by JVM for performance optimization

### 20. Why is String immutable in Java?
- **Thread safety**: Multiple threads can safely share String objects without synchronization
- **Security**: Protects sensitive data like passwords
- **Performance**: Enables String pooling, caching, and optimization
- **Class loading**: Ensures class names remain consistent during class loading
- **Garbage collection**: Reduces GC overhead
- Reference: https://youtu.be/to9DPVsdByE

### 21. What is the difference between String, StringBuffer, and StringBuilder?
- **String**: Immutable, thread-safe, slower for concatenation
- **StringBuffer**: Mutable, synchronized (thread-safe), slower than StringBuilder
- **StringBuilder**: Mutable, not synchronized (not thread-safe), faster for concatenation
- **Use case**: Use StringBuilder for single-threaded scenarios; StringBuffer for multithreading

### 22. How to create an immutable class?
- Declare class as `final` (cannot be subclassed)
- Declare all variables as `private final`
- No setter methods
- Provide parametrized constructor for deep copy
- Return cloned objects from getter methods if returning mutable objects

### 23. What are the common String methods?
- length(), isEmpty(), equals(), equalsIgnoreCase()
- startsWith(), endsWith(), contains()
- charAt(), indexOf(), lastIndexOf()
- substring(), split(), replace()
- toUpperCase(), toLowerCase(), trim()
- concat(), compareTo(), compareTo()

### 24. What is the difference between equals() and == in String comparison?
- **==**: Compares object references (memory addresses)
- **equals()**: Compares content/values of objects
- **For String**: Use equals() for content comparison, == for reference comparison
- **Example**: `"abc" == new String("abc")` is false (different references)
- **Example**: `"abc".equals(new String("abc"))` is true (same content)

### 25. How to find duplicate strings or remove duplicates from a list?
- **Using Set**: `new HashSet<>(list)` removes duplicates
- **Using Stream API (Java 8)**: `list.stream().distinct().collect(Collectors.toList())`
- **Using Set in loop**: Add to Set while iterating; Set automatically handles duplicates

### 26. What are the wrapper classes in Java?
- Primitive wrapper classes: Integer, Long, Double, Float, Boolean, Byte, Short, Character
- Purpose: To convert primitives into objects for use in collections, synchronization, multithreading
- **Autoboxing**: Automatic conversion of primitive to wrapper (int → Integer)
- **Unboxing**: Automatic conversion of wrapper to primitive (Integer → int)

### 27. What is the difference between Comparable and Comparator?
- **Comparable**: Implemented by class to sort objects (implements `Comparable<T>`, add compareTo() method)
- **Comparator**: External implementation for custom sorting (implements `Comparator<T>`, add compare() method)
- **Example**: Sort by name, age, or custom logic separately using Comparator

### 28. What is the difference between indexOf() and contains()?
- **indexOf()**: Returns index position of first occurrence or -1 if not found
- **contains()**: Returns boolean (true/false) if element exists

### 29. What is reflection API in Java?
- Allows inspection and modification of classes, methods, fields at runtime
- **Methods**: getClassName(), forName(), .class, newInstance()
- **Cons**: Breaks encapsulation principle; allows access to private methods/variables
- **Use case**: Frameworks like Spring, Hibernate use reflection for dependency injection

### 30. What is the difference between final, finally, and finalize?
- **final**: Keyword for variables (immutable), methods (cannot override), classes (cannot extend)
- **finally**: Block in exception handling that always executes (cleanup code)
- **finalize**: Method called by garbage collector before object destruction; used for cleanup activities

### 31. What is the this keyword in Java?
- Reference variable pointing to current object
- Used to refer to instance variables and methods of current object
- Used in constructor chaining to call other constructors

### 32. What is the super keyword in Java?
- Used to refer to parent class
- Used to call parent class constructor
- Used to call parent class methods that are overridden in child class

### 33. What is the instanceof operator?
- Used to check if an object is instance of a class or interface
- Returns boolean (true/false)
- Syntax: `object instanceof ClassName`

### 34. What are the access modifiers in Java?
- **public**: Accessible from anywhere
- **private**: Accessible only within the class
- **protected**: Accessible within package and subclasses
- **default (no modifier)**: Accessible within package only

### 35. Can we declare a top-level class as private?
- **No**: Top-level classes can only be public or default
- **Yes**: Inner classes can be declared as private (cannot be accessed from outside the outer class)

### 36. What is call by value and call by reference?
- **Java uses only call by value**: A copy of value is passed to method
- **For primitives**: Changes inside method don't affect original value
- **For objects**: Copy of reference is passed; changes to object properties are reflected outside
- **Why no pointers in Java**: For security, simplicity, and garbage collection

### 37. What is the difference between static and non-static members?
- **static**: Belongs to class; shared among all instances; initialized once at class loading
- **non-static**: Belongs to instance; each object has its own copy; initialized for each object
- **Memory**: Static variables stored in method area; instance variables in heap

### 38. What is the difference between default and static methods in interface?
- **default method**: Instance method with implementation; can be overridden
- **static method**: Cannot be overridden; called on interface directly
- **Access**: default methods accessible through object; static methods through interface

### 39. What are initialization blocks in Java?
- **static block**: `static { }` - Executed once at class loading time; used for static data initialization
- **instance block (non-static)**: `{ }` - Executed each time object is created; used for instance initialization

### 40. What is constructor chaining?
- Calling one constructor from another within same class
- Syntax: `this(args)` to call another constructor
- Throws chained exceptions for error handling

### 41. What is shadowing in Java?
- When a local variable inside a block overrides an outer scope variable with same name
- Local variable shadows the outer scope variable within that block

### 42. What is the difference between shallow copy and deep copy?
- **Shallow copy**: Copies references of objects, not actual objects themselves
- **Deep copy**: Copies actual objects; independent copy of all nested objects
- **Use**: Clone objects using clone() method or custom copy constructor

## Object-Oriented Programming (OOPs)

### 43. Explain the four pillars of OOP
- **Encapsulation**: Bundling data and methods; hiding internal details through access modifiers
- **Abstraction**: Hiding complexity; showing only essential features
- **Inheritance**: IS-A relationship; extending parent class functionality
- **Polymorphism**: One interface, multiple implementations; method overloading/overriding

### 44. What is inheritance?
- IS-A relationship between classes
- Child class inherits properties and methods from parent class
- Types: Single, multi-level, hierarchical (not multiple in Java, but through interfaces)

### 45. What is the diamond problem?
- Occurs with multiple inheritance when B, C extend A; then D extends both B and C
- Java resolves it using default interface methods (Interface 1 and Interface 2 can have same method)
- Resolution: Implement interface methods in class or use super keyword to specify which interface method to call

### 46. What is polymorphism?
- One interface, multiple implementations
- **Compile-time (Static) Polymorphism**: Method overloading
- **Runtime (Dynamic) Polymorphism**: Method overriding; also called Dynamic Method Dispatch
- Enables flexibility and loose coupling

### 47. What is method overloading?
- Same method name with different parameters in same class
- Differences: Number of parameters, type of parameters, or order of parameters
- **Static Polymorphism**: Decision made at compile time
- **Example**: public void add(int a, int b) and public void add(double a, double b)

### 48. What is method overriding?
- Child class provides different implementation for parent class method
- Signature must be same (method name, parameters, return type)
- **Dynamic Polymorphism**: Decision made at runtime
- **Can override static methods**: No; static methods cannot be overridden

### 49. Can we override static methods?
- **No**: Static methods cannot be overridden
- **Reason**: Static methods are bound at compile time, not runtime
- **Can hide static method**: Child class can declare same static method (method hiding, not overriding)

### 50. What is abstraction?
- Hiding complexity; showing only essential features
- **Abstract class**: Cannot instantiate; may contain abstract and concrete methods
- **Interface**: Contract for implementing classes; all methods are abstract (before Java 8)
- **Purpose**: Define blueprint for subclasses; enforce implementation standards

### 51. What is the difference between abstract class and interface?
- **Abstract class**: Can have state (variables with values); can have constructors; can be extended by any class
- **Interface**: Only has contracts (abstract methods); no state; no constructors (before Java 8); multiple inheritance possible
- **Methods**: Abstract class can have concrete methods; interface all methods abstract (before Java 8)
- **Access modifiers**: Abstract class members can be private, protected; interface members public only (before Java 8)
- **Java 8+**: Interfaces can have default methods and static methods

### 52. Can we have a constructor in an abstract class?
- **Yes**: Abstract class can have constructors
- **Purpose**: To initialize parent class state; called via super() from child class
- **Note**: Public constructor in abstract class is useless since we cannot instantiate abstract class

### 53. Can we have a constructor in an interface?
- **No**: Interfaces cannot have constructors
- **Reason**: No state/variables defined in interface; no object created

### 54. What are marker interfaces?
- Interfaces with no methods; used to mark a class as having special meaning
- **Examples**: Cloneable, Serializable
- **Purpose**: Indicate to JVM or framework that marked class has special behavior
- **No implementation required**: Just implementing interface is enough

### 55. What is a functional interface?
- Interface with exactly one abstract method
- **Examples**: Runnable, Comparator, Consumer, Supplier, Predicate
- **Java 8+**: Used with lambda expressions for concise code
- **@FunctionalInterface**: Annotation to ensure interface has only one abstract method

### 56. What is encapsulation?
- Bundling data and methods together
- Hiding internal details through access modifiers
- Achieved using private variables and public getter/setter methods
- **Benefits**: Security, flexibility in implementation, easy maintenance

### 57. What is coupling?
- Dependency between objects or modules
- **Loose coupling**: Modules depend on abstractions (interfaces), not concrete implementations
- **Tight coupling**: Modules directly depend on concrete classes
- **How to achieve loose coupling**: Use interfaces, dependency injection

### 58. What is cohesion?
- Degree to which components of a class or module work together
- **High cohesion**: Class has single well-defined responsibility
- **Low cohesion**: Class has mixed unrelated responsibilities
- **Example**: java.io (file operations) has high cohesion; java.util (collections) has high cohesion

### 59. What is association?
- Relationship between objects/classes
- **Types**: 1-to-1, 1-to-Many, Many-to-Many, Many-to-1

### 60. What is aggregation?
- HAS-A relationship; weak association
- Object contains other objects as part of its state
- Contained objects can exist independently
- **Example**: Department has Employees; if Department is deleted, Employees can still exist

### 61. What is composition?
- Strong HAS-A relationship; objects are tightly coupled
- If parent object is deleted, all child objects are automatically deleted
- **Example**: Car has Engine; if Car is deleted, Engine is also deleted

### 62. Does Java support multiple inheritance?
- **Direct multiple inheritance**: No
- **Through interfaces**: Yes; a class can implement multiple interfaces
- **Resolution**: Resolved using default interface methods (Java 8+)

## Java Collections & Data Structures

### 63. What is the Collections framework in Java?
- Set of classes and interfaces for storing and manipulating groups of objects
- **Main interfaces**: Collection, Set, List, Queue, Map
- **Implementations**: ArrayList, LinkedList, HashSet, HashMap, TreeSet, TreeMap

### 64. What is the difference between Collection and Collections?
- **Collection**: Interface in java.util package; root interface for all collection types
- **Collections**: Utility class with static methods for collection operations (sort, shuffle, binarySearch, etc.)

### 65. What is the difference between Array and ArrayList?
- **Array**: Fixed size; can contain primitives and objects; not part of collections framework
- **ArrayList**: Dynamic size; can only contain objects; part of collections framework; implements List interface
- **Performance**: Array access is faster; ArrayList provides convenience methods

### 66. What is the default capacity of ArrayList?
- Initial capacity is 10
- When capacity is exceeded, it grows by 1.5x (new_capacity = old_capacity * 1.5)
- Internally uses Object[] array

### 67. What is the difference between ArrayList and LinkedList?
- **ArrayList**: Backed by dynamic array; fast random access (O(1)); slow insertion/deletion in middle (O(n))
- **LinkedList**: Backed by doubly linked list; slow random access (O(n)); fast insertion/deletion at ends (O(1))
- **Memory**: ArrayList has less memory overhead; LinkedList has more (stores previous and next node references)
- **Use case**: ArrayList for frequent retrieval; LinkedList for frequent insertion/deletion at ends

### 68. What is the difference between Set and List?
- **List**: Ordered collection; allows duplicates; maintains insertion order; access by index
- **Set**: Unordered collection; no duplicates; does not maintain insertion order; no index-based access
- **Types of Set**: HashSet (unordered), TreeSet (sorted), LinkedHashSet (insertion order)

### 69. What is HashMap?
- Key-Value pair collection; implements Map interface
- Uses hash function to store and retrieve values
- Not thread-safe; not synchronized
- Allows one null key and multiple null values
- **Time complexity**: O(1) average for get, put, remove

### 70. How does HashMap internally work?
- Uses array of buckets (Node[] table)
- Hash function converts key to index: `index = hashCode(key) % table.length`
- **Collision handling**: Uses chaining (linked list of nodes) to handle collisions
- **Before Java 8**: Collisions handled with linked lists; complexity O(n) in worst case
- **Java 8+**: Collisions handled with balanced trees (Red-Black tree) when bucket has 8+ entries; complexity O(log n)

### 71. What is the default capacity of HashMap?
- Initial capacity is 16
- Load factor is 0.75 (when 75% capacity is filled, HashMap resizes)
- New capacity = old capacity * 2

### 72. Can HashMap have null keys?
- **Yes**: HashMap can have one null key
- Hash code for null key is 0 by convention
- Null key is stored at index 0

### 73. Can HashMap have multiple null keys?
- **No**: Only one null key allowed
- Storing another null key overwrites the previous null key entry

### 74. Can HashMap have multiple null values?
- **Yes**: HashMap can have multiple null values
- Multiple keys can map to null values

### 75. What is the difference between HashMap and Hashtable?
- **Hashtable**: Thread-safe (synchronized); legacy class (before Java 5)
- **HashMap**: Not thread-safe; newer implementation; faster than Hashtable
- **Null keys/values**: Hashtable doesn't allow null keys or values; HashMap allows both
- **Performance**: HashMap is faster; use ConcurrentHashMap for thread-safe alternative

### 76. What is ConcurrentHashMap?
- Thread-safe implementation of HashMap
- Uses bucket-level locking (segment locks); multiple threads can access different buckets simultaneously
- Better performance than Hashtable for concurrent access
- Does not allow null keys or values

### 77. What is HashSet?
- Unordered collection of unique elements
- Uses HashMap internally (HashMap<E, Object>)
- No duplicates allowed
- **Time complexity**: O(1) average for add, remove, contains

### 78. What is the difference between HashMap and HashSet?
- **HashMap**: Stores key-value pairs; allows null key and values
- **HashSet**: Stores unique elements; allows only one null value
- **Implementation**: HashSet internally uses HashMap with dummy Object values

### 79. What is the synchronized keyword?
- Ensures only one thread can execute the method/block at a time
- Prevents race conditions in multithreading
- **Synchronized method**: Locks entire object
- **Synchronized block**: Locks specific object or resource

### 80. What is TreeSet and TreeMap?
- **TreeSet**: Sorted set; implements SortedSet interface; elements sorted using natural order or comparator
- **TreeMap**: Sorted map; implements SortedMap interface; keys sorted using natural order or comparator
- **Implementation**: Backed by Red-Black tree
- **Time complexity**: O(log n) for operations
- **Use case**: When sorted order is required

### 81. What is the hashCode() method?
- Returns integer hash code for an object
- Used internally by hash-based collections (HashMap, HashSet, Hashtable)
- **Contract**: If two objects are equal (using equals()), they must have same hash code
- Override hashCode() when overriding equals() method

### 82. What are the most used methods in Collections class?
- sort() - Sort collection in natural or custom order
- shuffle() - Randomly shuffle collection elements
- reverse() - Reverse collection order
- binarySearch() - Search element in sorted collection
- min()/max() - Find minimum/maximum element
- addAll() - Add all elements from one collection to another
- frequency() - Count occurrences of element
- disjoint() - Check if two collections have no common elements

### 83. What is the difference between Stream and Collections?
- **Collections**: Store data in memory; in-memory data structure
- **Stream**: No storage; functional programming approach; processes data on-the-fly
- **Modification**: Collections allow modification; Streams are immutable
- **Laziness**: Collections are eager; Streams are lazy (process elements on-demand)

## Multithreading & Concurrency

### 84. What is the difference between process and thread?
- **Process**: Independent program in execution; has its own memory space; heavy-weight
- **Thread**: Lightweight unit of process; threads within same process share memory space; light-weight
- **Context switching**: Expensive for processes; cheap for threads

### 85. What is multithreading?
- Running multiple threads simultaneously within same process
- Allows CPU to switch between threads, giving illusion of parallel execution
- **On multi-core systems**: Threads execute truly in parallel

### 86. What is the difference between process-based and thread-based multitasking?
- **Process-based**: Multiple processes run simultaneously; heavy-weight; expensive context switching
- **Thread-based**: Multiple threads within same process; light-weight; cheap context switching; share memory
- **Java uses**: Thread-based multitasking

### 87. How to create a thread in Java?
- **Method 1**: Extend Thread class and override run() method
- **Method 2**: Implement Runnable interface and override run() method (preferred)
- **Method 2 advantage**: Can extend another class while implementing Runnable

### 88. What is the difference between extending Thread class and implementing Runnable?
- **Extend Thread**: 
  - Direct subclass of Thread
  - Only one parent class possible (no multiple inheritance)
  - Access Thread methods directly
- **Implement Runnable** (Preferred):
  - Runnable is interface
  - Can extend another class while implementing Runnable
  - Better design (composition over inheritance)
  - More flexible

### 89. How to run a thread in Java?
- Create Thread object: `Thread t = new Thread(runnable_object)`
- Call start() method: `t.start()`
- **Never call run()**: Direct run() call executes method in current thread, not in new thread

### 90. What happens when we call start() method twice on same thread?
- Throws **IllegalThreadStateException**
- Thread can be started only once

### 91. What are the different states of a thread?
- **New**: Thread created but not started
- **Runnable**: Thread started; ready to run or running
- **Blocked**: Thread waiting for resource/lock
- **Waiting**: Thread waiting for notification from another thread
- **Timed Waiting**: Thread waiting for timeout or notification
- **Terminated**: Thread execution completed

### 92. What is thread priority?
- Integer value from 1 to 10 (default is 5 - NORM_PRIORITY)
- MIN_PRIORITY = 1, NORM_PRIORITY = 5, MAX_PRIORITY = 10
- Higher priority thread gets more CPU time
- **Set priority**: `thread.setPriority(8)`
- **Get priority**: `thread.getPriority()`

### 93. What is the join() method in threads?
- Makes current thread wait for another thread to complete execution
- **Syntax**: `thread.join()`
- **Use case**: Ensure thread completes before main thread proceeds
- **Throws**: InterruptedException

### 94. What is a daemon thread?
- Background thread that does not prevent JVM from exiting
- JVM exits when all non-daemon threads finish
- **Set daemon**: `thread.setDaemon(true)` (must be called before start())
- **Example**: Garbage collector, time-based tasks

### 95. What is synchronization of threads?
- Mechanism to ensure only one thread accesses a resource at a time
- Prevents race conditions and data inconsistency
- **Methods**:
  - Synchronized method
  - Synchronized block
  - Using volatile keyword

### 96. What is the synchronized keyword?
- Used to lock object or class to prevent concurrent access
- **Synchronized method**: Entire method is locked
- **Synchronized block**: Only specific code section is locked
- **Lock**: Implicit lock on object (method) or specified object (block)

### 97. Can a static method be synchronized?
- **Yes**: Static method can be synchronized
- **Lock**: Locks class-level (all threads share class lock), not object-level
- Only one thread can access synchronized static method at a time across all instances

### 98. What is a deadlock?
- Situation where two or more threads are waiting indefinitely for each other
- **Occurs when**: Thread A holds lock on Resource 1 and waits for Resource 2; Thread B holds Resource 2 and waits for Resource 1
- **Consequences**: Application freezes; no progress

### 99. How to avoid deadlocks?
- Acquire locks in same order across all threads
- Use timeout on lock acquisition
- Use try-finally to release locks
- Avoid nested locks
- Use concurrent collections and higher-level abstractions

### 100. What is the use of wait() method?
- Called by thread to release lock and wait for another thread's notification
- Thread enters waiting state; released lock is acquired by another thread
- Must be called within synchronized context
- **Throws**: InterruptedException

### 101. What is the use of notify() method?
- Notifies one waiting thread to wake up
- Awakened thread must reacquire lock before proceeding
- Other waiting threads remain waiting
- Must be called within synchronized context

### 102. What is the use of notifyAll() method?
- Notifies all waiting threads to wake up
- All awaiting threads enter runnable state; only one acquires lock
- Others wait for lock release
- Must be called within synchronized context

### 103. What is the volatile keyword?
- Indicates variable can be modified by multiple threads
- Prevents compiler optimizations that might cache variable value
- **Java Memory Model**: Ensures visibility across threads (happens-before relationship)
- Slower than non-volatile (no caching)
- Does NOT provide atomicity

### 104. What is the difference between volatile and synchronized?
- **volatile**: Only ensures visibility; does NOT provide atomicity
- **synchronized**: Provides both visibility and atomicity (mutual exclusion)
- **Performance**: volatile is faster; synchronized is slower

### 105. What is an atomic operation?
- Operation that completes without interruption
- Entire operation treated as single unit; cannot be split
- **Example**: AtomicInteger provides thread-safe increment without synchronization
- **Classes**: AtomicInteger, AtomicLong, AtomicBoolean, AtomicReference

### 106. What is ExecutorService?
- Interface for managing pool of threads
- Simplifies thread creation and management
- Executes tasks asynchronously
- **Methods**: execute(), submit(), shutdown(), shutdownNow()

### 107. How to create ExecutorService?
- Using Executors factory methods:
  - `Executors.newFixedThreadPool(int nThreads)` - Fixed thread pool
  - `Executors.newCachedThreadPool()` - Creates threads as needed
  - `Executors.newSingleThreadExecutor()` - Single thread
  - `Executors.newScheduledThreadPool(int corePoolSize)` - Scheduled tasks

### 108. What is Callable in threading?
- Similar to Runnable; returns result of computation
- **Runnable.run()**: Returns void
- **Callable.call()**: Returns result; can throw checked exception
- Used with ExecutorService.submit() method
- Returns Future object containing result

### 109. What is Future in Java?
- Represents result of asynchronous computation
- Methods: get() (blocking), isDone(), isCancelled(), cancel()
- **get()**: Waits for result; throws ExecutionException if task failed

### 110. How to check if ExecutorService task executed successfully?
- Submit task using submit() which returns Future
- Check using `future.isDone()` - returns true if task completed
- Get result using `future.get()` - returns result or throws exception if failed
- Catch ExecutionException to handle task failures

### 111. What is thread pool?
- Collection of pre-created threads ready to execute tasks
- **Advantages**: Reduces thread creation overhead; reuses threads; controls thread count
- **Components**: Queue of tasks; pool of threads; thread manager

### 112. What is the thread lifecycle diagram?
```
New → Runnable → Running → Blocked/Waiting → Runnable → Terminated
```

## Exception Handling

### 113. What is the difference between error and exception?
- **Error**: Serious problem that application cannot recover from (StackOverflowError, OutOfMemoryError)
- **Exception**: Condition that application can handle and recover from

### 114. What are checked exceptions?
- Exceptions checked at compile time
- Must be caught or declared (throws) in method signature
- **Examples**: IOException, SQLException, ClassNotFoundException, FileNotFoundException
- **Inheritance**: Inherit from Exception class (excluding RuntimeException)

### 115. What are unchecked exceptions?
- Exceptions checked at runtime, not compile time
- Do NOT need to be caught or declared
- **Examples**: NullPointerException, ArrayIndexOutOfBoundsException, ArithmeticException, ClassCastException
- **Inheritance**: Inherit from RuntimeException

### 116. What is the difference between throw and throws?
- **throw**: Used to throw specific exception explicitly; followed by exception instance
  - `throw new IllegalArgumentException("Invalid argument");`
- **throws**: Used in method signature to declare exceptions thrown by method
  - `public void method() throws IOException, SQLException { }`

### 117. What is the try-catch-finally block?
- **try**: Code that might throw exception
- **catch**: Handles exception if thrown
- **finally**: Block that always executes (cleanup code)
- **Order**: catch Exception before Checked Exceptions; finally always executes (even if return in try/catch)

### 118. What is exception chaining?
- Wrapping one exception inside another to provide context
- **Syntax**: `throw new CustomException("message", cause);`
- **Purpose**: Preserve original exception while providing additional context
- Used in constructor chaining for error handling

### 119. What is try-with-resources?
- Feature introduced in Java 7
- Automatically closes resources implementing AutoCloseable interface
- Syntax: `try (ResourceType resource = new Resource()) { }`
- **Advantage**: No need for finally block to close resources

### 120. How to handle multiple exception types?
- Catch most specific exception first, then general exceptions
- **Java 7+**: Multi-catch: `catch (IOException | SQLException e) { }`
- **Order matters**: Catch Exception last (most general)

### 121. How to create custom exception?
- Extend Exception class (for checked exception) or RuntimeException (for unchecked)
- Override constructor to accept message and cause
- ```java
  public class CustomException extends Exception {
    public CustomException(String message) { super(message); }
    public CustomException(String message, Throwable cause) { super(message, cause); }
  }
  ```

### 122. What design pattern is used for exception handling?
- **Chain of Responsibility Pattern**: Each catch block is a handler; exception passes through chain until handled

## Java 8+ Features

### 123. What are the major features of Java 8?
- Lambda expressions
- Stream API
- Functional interfaces
- Default methods in interfaces
- Static methods in interfaces
- Method references
- Optional class
- forEach() method
- Base64 encoding/decoding

### 124. What is a lambda expression?
- Anonymous function with no name
- **Syntax**: `(parameters) -> body`
- **Examples**: 
  - `(x, y) -> x + y`
  - `name -> System.out.println(name)`
  - `() -> System.out.println("Hello")`
- **Use with**: Functional interfaces
- **Advantage**: Concise code; reduced boilerplate

### 125. What are functional interfaces?
- Interface with exactly one abstract method
- **@FunctionalInterface** annotation (optional but recommended)
- **Examples**: Runnable, Callable, Comparator, Consumer, Supplier, Predicate, Function
- **Java 8+**: Can have default and static methods in addition to abstract method

### 126. What is the difference between different functional interfaces?
- **`Consumer<T>`**: Accepts argument, returns void; `accept(T t)`
- **`Supplier<T>`**: No argument, returns value; `get()` → T
- **`Predicate<T>`**: Accepts argument, returns boolean; `test(T t)` → boolean
- **`Function<T, R>`**: Accepts T, returns R; `apply(T t)` → R

### 127. What is the Stream API?
- Functional programming approach to process collections
- **Characteristics**:
  - No storage; processes data on-the-fly
  - Functional programming model
  - Possibly unbounded
  - Laziness-seeking (processes only needed elements)
- **Creation**: `collection.stream()` or `Stream.of()`
- **Operations**: map, filter, reduce, collect, forEach

### 128. What are intermediate and terminal operations in Stream?
- **Intermediate operations**: Return Stream; lazy evaluation
  - map(), filter(), distinct(), limit(), skip(), sorted()
- **Terminal operations**: Trigger evaluation; return non-stream result
  - forEach(), collect(), reduce(), count(), min(), max(), findFirst(), anyMatch()

### 129. What is the difference between map and flatMap in Stream?
- **map()**: Transforms each element to another; flattens by one level
  - `Stream<String> words = Stream.of("hello", "world")`
  - `words.map(String::toUpperCase)` → `Stream<String>`
- **flatMap()**: Transforms each element to Stream; flattens entire nested structure
  - `Stream<Stream<Integer>> nested = Stream.of(Stream.of(1,2,3), Stream.of(4,5,6))`
  - `nested.flatMap(s -> s)` → `Stream<Integer> [1,2,3,4,5,6]`

### 130. What is filter in Stream API?
- Intermediate operation; retains elements matching predicate
- **Syntax**: `stream.filter(predicate)` where `Predicate<T>` returns boolean
- **Example**: `list.stream().filter(x -> x > 5).collect(Collectors.toList())`

### 131. What is reduce in Stream API?
- Terminal operation; combines all elements into single result
- **Variants**:
  - `reduce(T identity, BinaryOperator<T> accumulator)` - Returns T
  - `reduce(BinaryOperator<T> accumulator)` - Returns `Optional<T>`
- **Example**: `list.stream().reduce(0, (a, b) -> a + b)` - Sum of elements

### 132. What is collect in Stream API?
- Terminal operation; combines elements into container (List, Set, Map)
- **Syntax**: `stream.collect(collector)`
- **Collectors class**: Provides common collectors (toList, toSet, toMap, groupingBy, joining, etc.)

### 133. What is the Collectors class?
- Utility class with static methods for collecting stream results
- **Common collectors**:
  - toList() - Collect to List
  - toSet() - Collect to Set
  - toMap(keyMapper, valueMapper) - Collect to Map
  - joining(delimiter) - Join elements to String
  - groupingBy(classifier) - Group elements by classifier
  - partitioningBy(predicate) - Partition into true/false groups

### 134. What is the difference between sequential and parallel streams?
- **Sequential stream**: Processes elements one by one; faster for small datasets
- **Parallel stream**: Processes elements in parallel using multiple threads; faster for large datasets
- **Creation**: `collection.parallelStream()` or `stream.parallel()`
- **When to use**: Parallel for large datasets with independent operations

### 135. What are method references?
- Shorthand notation for lambda expressions
- **Syntax**: `Class::method` or `object::method`
- **Types**:
  - Static method reference: `Math::abs`
  - Instance method reference: `String::toUpperCase`
  - Constructor reference: `String::new`

### 136. What is the Optional class?
- Container for optional value; might or might not contain value
- **Purpose**: Avoid null pointer exceptions
- **Methods**: isPresent(), isEmpty(), get(), orElse(), orElseThrow(), map(), filter(), flatMap()
- **Example**: `Optional<String> optional = Optional.of("value")`

### 137. What is the difference between Optional.of() and Optional.ofNullable()?
- **of()**: Throws NullPointerException if value is null
- **ofNullable()**: Returns empty Optional if value is null; returns Optional with value if not null

### 138. What are the major features of Java 9?
- Platform Module System (Project Jigsaw)
- Interface private methods
- Stream API improvements
- Enhanced try-with-resources
- JShell (REPL for Java)
- Factory methods for Collections (List.of(), Set.of(), Map.of())
- Process API improvements

### 139. What are the major features of Java 11?
- Local variable type inference (var keyword)
- String methods (isBlank(), lines(), strip())
- Collection.toArray() enhancement
- ChaCha20-Poly1305 cipher algorithms
- HTTP/2 support

### 140. Is Java paid after Java 11?
- **Oracle JDK**: Paid commercial license after Java 11 (subscription model)
- **OpenJDK**: Free and open-source under GPL license
- **LTS (Long Term Support)**: Java 8, 11, 17, 21 (free or paid depending on vendor)

### 141. What is the improvement in HashMap from Java 7 to Java 8?
- **Java 7**: Collision handling using linked list; worst-case complexity O(n)
- **Java 8**: Collision handling using balanced Red-Black tree when bucket size > 8; complexity O(log n)
- **Threshold**: When linked list size exceeds 8, converts to tree; when size falls to 6, converts back to linked list

## Spring Framework

### 142. What is Spring Framework?
- Open-source framework for building enterprise Java applications
- **Core features**: IoC container, dependency injection, AOP, transaction management
- **Modules**: Core, AOP, Data Access, Web, Integration, Security

### 143. What is Inversion of Control (IoC)?
- Design principle where control of object creation and lifecycle passed to container
- **Traditional approach**: Application creates objects
- **IoC approach**: Container creates and manages objects
- **Advantage**: Loose coupling, better testability, flexible configuration

### 144. What is Dependency Injection (DI)?
- Technique to provide object its dependencies
- **Types**:
  - Constructor injection
  - Setter injection
  - Field injection (using @Autowired)
- **Advantage**: Objects decoupled from dependencies; easier testing with mocks

### 145. What is the Spring IoC container?
- Core component of Spring Framework
- **Responsibilities**: Create objects, manage lifecycle, inject dependencies
- **Two main interfaces**:
  - **BeanFactory**: Basic IoC container
  - **ApplicationContext**: Advanced IoC container (preferred)

### 146. What is ApplicationContext?
- Interface extending BeanFactory
- **Responsibilities**: Manage bean lifecycle, support event propagation, internationalization
- **Implementations**: ClassPathXmlApplicationContext, FileSystemXmlApplicationContext, AnnotationConfigApplicationContext

### 147. What is DispatcherServlet?
- Central servlet in Spring MVC
- **Responsibilities**: Receive HTTP requests, route to appropriate controller, send response
- **Flow**: Request → DispatcherServlet → Handler Mapping → Controller → View Resolver → Response

### 148. What is the bean lifecycle in Spring?
1. Instantiation: Bean instance created
2. Populate properties: Dependency injection
3. BeanNameAware.setBeanName()
4. BeanFactoryAware.setBeanFactory()
5. Pre-initialization: @PostConstruct or InitializingBean.afterPropertiesSet()
6. Custom initialization
7. Post-initialization: BeanPostProcessor.postProcessAfterInitialization()
8. In use
9. Destruction: @PreDestroy or DisposableBean.destroy()

### 149. What are bean scopes in Spring?
- **singleton**: Single instance per Spring container (default)
- **prototype**: New instance for each request
- **request**: New instance for each HTTP request (web applications)
- **session**: New instance for each HTTP session (web applications)
- **global-session**: New instance for global session (portlet applications)

### 150. How to configure beans in Spring?
- **XML configuration**: Define beans in XML file
- **Annotation-based**: Use @Component, @Service, @Repository, @Controller, @Bean
- **Java-based**: @Configuration class with @Bean methods (preferred)

### 151. What is the difference between XML and annotation-based configuration?
- **XML**: Centralized configuration; separation of concerns; verbose
- **Annotation-based**: Compact code; less configuration; more intuitive
- **Modern approach**: Annotation-based preferred

### 152. What is @Component annotation?
- Marks class as Spring-managed component
- **Purpose**: Automatic bean creation and injection
- **Variants**: @Service (business logic), @Repository (data access), @Controller (presentation layer)

### 153. What is the difference between @Component, @Service, and @Repository?
- **@Component**: Generic component
- **@Service**: Business logic layer; semantic meaning
- **@Repository**: Data access layer; provides exception translation
- **@Controller**: Presentation layer; handles requests

### 154. What is @Bean annotation?
- Marks method to produce bean managed by Spring container
- Used in @Configuration classes
- **Purpose**: Create beans for third-party libraries or complex configurations
- **Example**: `@Bean public DataSource dataSource() { return new DataSource(); }`

### 155. What is @Autowired annotation?
- Automatically injects dependency
- **Works on**: Field, constructor, setter method
- **Behavior**: By type; if multiple beans, use @Qualifier or @Primary
- **Optional injection**: `@Autowired(required = false)`

### 156. What is the difference between @Qualifier and @Primary?
- **@Primary**: Marks bean as default when multiple candidates exist
- **@Qualifier**: Specifies exact bean name to inject
- **Use case**: @Primary for one default; @Qualifier for specific selection

### 157. What is @Configuration annotation?
- Marks class as source of bean definitions
- Methods annotated with @Bean define beans
- **Advantage**: Type-safe, IDE support, easier to test

### 158. What is @ComponentScan annotation?
- Tells Spring where to find component classes
- **Syntax**: `@ComponentScan(basePackages = "com.example")`
- **Default**: Scans package of @Configuration class and subpackages

### 159. What is @EnableAutoConfiguration?
- Attempts to guess and configure Spring application based on jar dependencies
- Automatically configures DataSource, JPA, etc.
- **Shorthand**: @SpringBootApplication includes @EnableAutoConfiguration

### 160. What is AOP (Aspect-Oriented Programming)?
- Technique to modularize cross-cutting concerns (logging, security, transactions)
- **Concepts**: Aspect (module), Join point (method execution), Pointcut (where aspect applies), Advice (what to do)
- **Advantage**: Separation of concerns; reduces code duplication

### 161. What is the difference between Spring AOP and AspectJ?
- **Spring AOP**: Proxy-based; easier to use; limited to method-level Join points
- **AspectJ**: Byte-code manipulation; more powerful; supports field-level Join points
- **Use case**: Spring AOP for most cases; AspectJ for complex scenarios

### 162. What is Spring Boot?

### 163. What is the difference between Spring and Spring Boot?
- **Spring**: Framework with manual configuration required

### 164. What is auto-configuration in Spring Boot?

### 165. What is @SpringBootApplication?

### 166. How does Spring Boot bootstrap process work?
1. JVM starts → executes main() method

3. Loads properties from application.properties/yml

### 172. What is @RestController?

- **Methods**: Annotated with @GetMapping, @PostMapping, etc.

### 173. What is the difference between @RestController and @Controller?
- **@Controller**: Returns view name (ModelAndView)
- **@RestController**: Returns data directly (JSON/XML) via @ResponseBody
- **View resolution**: @Controller uses view resolver; @RestController bypasses

### 174. What is @RequestMapping?
- Maps HTTP requests to handler methods

- **Shorthand**: @GetMapping, @PostMapping, @PutMapping, @DeleteMapping, @PatchMapping

### 175. What is the difference between @GetMapping and @PostMapping?
- **@GetMapping**: Maps HTTP GET requests
- **@PostMapping**: Maps HTTP POST requests
- Shorthand for @RequestMapping(method = RequestMethod.GET/POST)

### 176. What is the difference between @PathVariable and @RequestParam?

- **@RequestParam**: Extracts parameter from query string
  - `@GetMapping("/users")` with `@RequestParam(name = "id") int id`
  - Example: GET /users?id=123 → id = 123

### 177. What is @RequestBody?
- Binds HTTP request body to method parameter

### 178. What is @ResponseBody?
- Binds method return value to HTTP response body
- Automatically serializes object to JSON/XML
- Implicit in @RestController

### 179. What is ResponseEntity?
- Represents HTTP response with status, headers, body
- **Advantages**: Control status code, headers, body together
- **Example**: `ResponseEntity<User> response = new ResponseEntity<>(user, HttpStatus.OK)`

### 180. What is DTO (Data Transfer Object)?
- Object carrying data between processes
- **Purpose**: Separate internal domain objects from API contracts
- **Advantage**: Flexibility to change internal structure without affecting API
- **Difference from Entity**: Entity represents database table; DTO represents API contract

### 181. How to read properties from application.properties?

### 182. What is the difference between application.properties and application.yml?

- **Functionality**: Same; just different formats
- Example:
  ```

## Hibernate & JPA

### 222. What is JPA (Java Persistence API)?
- Standard API for ORM (Object-Relational Mapping)
- **Purpose**: Map Java objects to database tables
- **Implementations**: Hibernate, TopLink, EclipseLink, OpenJPA

### 223. What is Hibernate?
- Most popular ORM framework; implements JPA
- **Purpose**: Bridge gap between object model and relational model
- **Features**: HQL queries, caching, lazy loading, transaction management

### 224. What is the difference between Hibernate and JPA?
- **JPA**: Specification/standard API
- **Hibernate**: Implementation of JPA (also offers extensions beyond JPA)
- **Relationship**: Hibernate implements JPA; not vice versa
- **Use**: Program to JPA interface; can switch implementations

### 225. What is ORM (Object-Relational Mapping)?
- Technique mapping database tables to Java objects
- **Mapping**: Table → Class, Row → Object, Column → Property
- **Advantage**: Write object-oriented code; framework handles database operations

### 226. What is @Entity annotation?
- Marks class as JPA entity
- **Mapping**: Entity class maps to database table
- Table name defaults to class name (customizable with @Table)

### 227. What is @Table annotation?
- Customizes table name and schema for entity
- **Syntax**: `@Table(name = "users", schema = "public")`
- **Default**: Table name = Entity class name

### 228. What is @Id annotation?
- Marks field as primary key
- **Requirement**: Every entity must have @Id
- **Types**: @Id alone or with @GeneratedValue

### 229. What is @GeneratedValue?
- Generates primary key value automatically
- **Strategies**:
  - AUTO (default) - Framework decides
  - SEQUENCE - Use database sequence
  - IDENTITY - Use auto-increment
  - TABLE - Use table for generation

### 230. What is @Column annotation?
- Customizes column name, length, nullable, etc.
- **Attributes**: name, nullable, unique, length
- **Example**: `@Column(name = "user_name", nullable = false, length = 100)`

### 231. What is @Transient annotation?
- Marks field to be excluded from persistence
- Field not mapped to any column
- **Use case**: Temporary/calculated fields

### 232. What is @JsonIgnore annotation?
- Excludes field from JSON serialization/deserialization
- Used in REST endpoints to hide sensitive data
- **Difference from @Transient**: @Transient is for JPA; @JsonIgnore is for JSON

### 233. What is relationship mapping in JPA?
- **Represents associations between entities**
- Types: One-to-One, One-to-Many, Many-to-One, Many-to-Many

### 234. What is @OneToOne mapping?
- One entity associated with one other entity
- **Example**: User has one Profile
- **Implementation**: Add @OneToOne annotation on field
- **Fetch type**: EAGER (default) or LAZY

### 235. What is @OneToMany mapping?
- One entity associated with many other entities
- **Example**: Department has many Employees
- **Implementation**: Add @OneToMany(mappedBy = "department") on collection field
- **Opposite**: Many-to-One on other side

### 236. What is @ManyToOne mapping?
- Many entities associated with one other entity
- **Example**: Many Employees belong to one Department
- **Implementation**: Add @ManyToOne annotation on field
- **Opposite**: One-to-Many on other side

### 237. What is @ManyToMany mapping?
- Many entities associated with many other entities
- **Example**: Students and Courses (each student takes multiple courses; each course has multiple students)
- **Implementation**: Requires join table
- **Owning side**: Uses @ManyToMany; inverse side uses mappedBy

### 238. What is mappedBy in JPA?
- Indicates inverse side of bidirectional relationship
- **Usage**: Non-owning side of relationship
- **Purpose**: Prevent duplicate columns in database
- **Relationship direction**: Owning side has foreign key; inverse uses mappedBy

### 239. What is lazy loading and eager loading?
- **Eager loading**: Load related entities immediately (default for @OneToOne, @ManyToOne)
- **Lazy loading**: Load related entities on demand (default for @OneToMany, @ManyToMany)
- **Performance**: Lazy loading better for performance; prevents loading unnecessary data
- **Implementation**: `@OneToMany(fetch = FetchType.LAZY)`

### 240. What is the difference between eager and lazy loading in pagination?
- **Eager loading with pagination**: Can cause N+1 query problem; joins multiple times
- **Lazy loading with pagination**: Separate queries for related data; can cause lazy loading exception if session closes
- **Solution**: Use @Transactional to keep session open; or use explicit joining in query

### 241. What is Querying in JPA?
- Retrieve data using JPQL or native SQL
- **JPQL**: Object-oriented query language (similar to SQL but for entities)
- **Native SQL**: Direct SQL queries
- **Methods**: createQuery(), createNamedQuery()

### 242. What is the difference between JPQL and native SQL?
- **JPQL**: Queries entities; database-independent; portable
- **Native SQL**: Raw SQL queries; database-specific; less portable
- **Performance**: Native SQL can be faster for complex queries
- **Use case**: JPQL for standard queries; native SQL for complex/database-specific

### 243. What is Criteria API?
- Type-safe query construction API
- **Advantage**: Compile-time checking; programmatic query building
- **Syntax**: More verbose than JPQL but type-safe
- **Use case**: Dynamic queries based on runtime conditions

### 244. What is named query?
- Pre-defined query attached to entity
- **Syntax**: `@NamedQuery(name = "User.findByName", query = "SELECT u FROM User u WHERE u.name = :name")`
- **Advantage**: Query defined once; reused multiple times; easy to maintain

### 245. How to override query caching in JPA?
- Use @QueryHints: `@QueryHints(value = @QueryHint(name = "org.hibernate.cacheable", value = "true"))`
- **Hibernate second-level cache**: Configure caching strategy
- **Cache providers**: EHCache, Infinispan, etc.

### 246. What is caching in JPA?
- **First-level cache**: Session-level cache; automatic
- **Second-level cache**: SessionFactory-level cache; shared across sessions
- **Purpose**: Improve performance by avoiding repeated database queries
- **Configuration**: Enable and configure in persistence.xml or Hibernate config

### 247. What is transaction management in JPA?
- **JDBC transactions**: Manual commit/rollback
- **JTA transactions**: Container-managed (application server)
- **Resource-local transactions**: @Transactional in Spring
- **Isolation levels**: READ_UNCOMMITTED, READ_COMMITTED, REPEATABLE_READ, SERIALIZABLE

### 248. How to fetch 1000 records from database efficiently?
- Use pagination: Fetch in chunks (e.g., 100 per page)
- Avoid N+1 query problem: Use JOIN FETCH or @EntityGraph
- Enable batch fetching: Hibernate batch_size property
- Consider lazy loading for large object graphs
- **Example**: `Page<User>` users = repo.findAll(PageRequest.of(0, 100))

### 249. How to fetch second-largest salary?
- Using SQL: `SELECT MAX(salary) FROM employees WHERE salary < (SELECT MAX(salary) FROM employees)`
- Using JPQL: `SELECT u FROM User u WHERE u.salary = (SELECT MAX(u2.salary) FROM User u2 WHERE u2.salary < (SELECT MAX(u3.salary) FROM User u3))`
- Using Stream API: Sort descending and get second element

### 250. Can we access multiple databases in single microservice?
- **Yes**: Configure multiple DataSources
- Create separate EntityManagerFactory for each database
- Use @Qualifier to specify which datasource to inject
- **Caution**: Distributed transactions across DBs are complex

## REST APIs & Web Services

### 251. What is a REST API?
- Interface following REST principles for accessing web services
- **Components**: Resources (entities), HTTP methods, representation (JSON/XML)
- **Stateless**: Each request contains all information needed

### 252. What are REST principles?
- **Client-server**: Separation of concerns
- **Stateless**: No session state stored on server
- **Cacheable**: Responses can be cached

- **Resource-oriented**: Resources identified by URIs

### 253. What is RESTful Web Service?
- Web service following REST principles
- **Characteristics**: Resource-based URIs, HTTP methods for operations, JSON/XML representation
- **Example**: GET /api/users (retrieve all users), POST /api/users (create user)

### 254. How to design a good REST API?

- Use HTTP methods correctly: GET (retrieve), POST (create), PUT (update), DELETE (delete)
- Use appropriate status codes: 200 OK, 201 Created, 400 Bad Request, 404 Not Found, 500 Server Error
- Version API: `/api/v1/users` for compatibility
- Provide meaningful error responses with error codes and messages

### 255. What are HTTP status codes?

### 256. How to pass multiple IDs to GET API using path variable?
- **Approach 1**: `/users/{id1}/related/{id2}` - Hierarchical
- **Approach 2**: `/users?ids=1,2,3` - Query parameter (preferred)
- Parse comma-separated IDs or use multiple parameters
- **Example**: `@GetMapping("/users") List<User> getUsers(@RequestParam List<Integer> ids)`

## Databases & SQL

### 264. What are ACID properties?
- **Atomicity**: Transaction all-or-nothing; either all operations succeed or all fail
- **Consistency**: Data in valid state before and after transaction
- **Isolation**: Transactions don't interfere with each other
- **Durability**: Committed data persists even on system failure

### 265. What are SQL JOIN types?
- **INNER JOIN**: Records with matching values in both tables
- **LEFT JOIN**: All records from left table + matching from right table
- **RIGHT JOIN**: All records from right table + matching from left table
- **FULL JOIN**: All records from both tables
- **CROSS JOIN**: Cartesian product of both tables

### 266. Does left join allow null values from right table?
- **Yes**: Left join returns all rows from left table; if no match in right table, right columns contain NULL

### 267. What is the difference between DELETE and TRUNCATE?
- **DELETE**: Removes rows; slower; can rollback; triggers fire
- **TRUNCATE**: Removes all rows; faster; cannot rollback (in auto-commit mode); no triggers
- **Identity**: TRUNCATE resets identity seed; DELETE doesn't

### 268. What are database indexes?
- Data structure for faster data retrieval
- **Trade-off**: Faster queries, slower inserts/updates
- **Types**: Primary key index, unique index, composite index, full-text index

### 269. What is primary key vs unique key?
- **Primary key**: Uniquely identifies record; not NULL; only one per table
- **Unique key**: Ensures unique values; can be NULL; multiple per table

### 270. What is foreign key?
- Column referencing primary key in another table
- **Purpose**: Maintain referential integrity
- **Constraint**: Prevents orphaned records

### 271. What are aggregation functions in SQL?
- COUNT(), SUM(), AVG(), MIN(), MAX()
- Used with GROUP BY clause
- **Example**: `SELECT department, AVG(salary) FROM employees GROUP BY department`

### 272. What is GROUP BY clause?
- Groups rows based on column values
- Used with aggregation functions
- **Example**: Find average salary by department

### 273. What is HAVING clause?
- Filters groups based on aggregate values
- Like WHERE but for groups
- **Difference**: WHERE filters rows; HAVING filters groups

### 274. What is stored procedure?
- Pre-compiled SQL code stored in database
- **Advantage**: Reusable, better performance, encapsulation
- **Disadvantage**: Database-specific, harder to version

### 275. What is trigger?
- Automatic SQL code executed on specific database events (INSERT, UPDATE, DELETE)
- **Use case**: Maintain audit trails, enforce business rules, update denormalized data

## Message Queues (RabbitMQ)

### 276. What is RabbitMQ?
- Message broker for asynchronous communication
- **Pattern**: Producer sends message → Broker queues → Consumer processes
- **Advantage**: Decoupling, asynchronicity, reliability

### 277. What is AMQP (Advanced Message Queuing Protocol)?
- Protocol for reliable messaging
- **Components**: Producers, Exchanges, Queues, Consumers
- **Implemented by**: RabbitMQ

### 278. What are exchanges in RabbitMQ?
- Receives messages from producers; routes to queues
- **Types**:
  - **Direct**: Exact match with routing key
  - **Fanout**: Sends to all bound queues
  - **Topic**: Pattern match with routing key
  - **Headers**: Match on message headers

### 279. What is exchange vs queue?
- **Exchange**: Receives messages; routes based on routing key
- **Queue**: Stores messages; consumers retrieve
- **Binding**: Connects exchange to queue

### 280. What is routing key?
- Determines which queue receives message
- Exchange uses routing key to decide destination
- **Example**: Direct exchange with routing_key="order.created" sends to specific queue

### 281. What is the difference between fanout and topic exchange?
- **Fanout**: Sends message to all bound queues regardless of routing key
- **Topic**: Routes based on routing key pattern; queues bind with patterns

### 282. What is Dead Letter Queue (DLQ)?
- Queue for messages that cannot be processed
- **Use case**: Handling failed messages
- **Configuration**: Requeue failed messages for retry or send to DLQ

### 283. How to implement RabbitMQ in Spring Boot?
- Add dependency: spring-boot-starter-amqp
- Configure connection in application.properties
- Create Producer class with @Autowired RabbitTemplate
- Create Consumer class with @RabbitListener on method
- Define Exchange, Queue, Binding in @Configuration class

## Search & Indexing (Elasticsearch)

### 284. What is Elasticsearch?
- Distributed search and analytics engine
- **Based on**: Lucene
- **Use case**: Full-text search, log analysis, metrics

### 285. What are documents and indices in Elasticsearch?
- **Document**: Unit of searchable data (similar to database row); contains fields (key-value pairs)
- **Index**: Collection of documents (similar to database table)
- **Mapping**: Schema defining field types

### 286. What are field types in Elasticsearch?
- Text (analyzed), Keyword (not analyzed), Integer, Long, Double, Boolean, Date
- **Text**: Suitable for full-text search (tokenized)
- **Keyword**: Exact match searches (not tokenized)

### 287. What is full-text search?
- Search functionality finding relevant documents based on search terms
- **Process**: Tokenization (split into words), stemming (reduce to root form), matching
- **Example**: Search "running" matches "run", "runs", "runner"

### 288. What is analysis in Elasticsearch?
- Process of converting text into tokens for searching
- **Components**: Character filter, Tokenizer, Token filter
- **Purpose**: Standardize and simplify search

### 289. What are aggregations in Elasticsearch?
- Summarize data; similar to SQL GROUP BY
- **Types**: Metrics (COUNT, SUM, AVG), Bucket (group by value)
- **Example**: Group documents by category; count by category

### 290. What is match vs filter query?
- **Match**: Full-text search query; scores relevance; slower
- **Filter**: Exact match; binary result; faster; cacheable
- **Use case**: Match for relevance-based search; filter for exact matching

### 291. What is must vs should in Elasticsearch?
- **must**: Clause must match (AND condition); filters
- **should**: Clause should match (OR condition); improves relevance
- **Example**: must=[status=active], should=[user=admin] (active documents, admin preferred)

### 292. What is term vs match query?
- **term**: Exact term match on non-analyzed field (keyword)
- **match**: Full-text search on analyzed field (text)
- **Example**: term on exact ID; match on search phrase

### 293. What are nested queries?
- Queries on nested objects within document
- **Use case**: Search on related objects (e.g., comments within article)
- **Syntax**: nested query with inner query on nested path

### 294. How to index documents in Elasticsearch?
- Use Elasticsearch client or REST API
- **REST API**: `PUT /index_name/_doc/1 { document }`
- **Implementation**: Use RestHighLevelClient in Spring Boot application
- **Batch**: Use bulk API for efficient indexing

## Design Patterns

### 329. What are SOLID principles?
- **Single Responsibility**: Class has one responsibility
- **Open/Closed**: Open for extension, closed for modification
- **Liskov Substitution**: Subclass can replace parent without breaking code
- **Interface Segregation**: Clients depend only on used methods
- **Dependency Inversion**: Depend on abstractions, not concrete implementations

### 330. What is singleton design pattern?
- Ensures class has only one instance; provides global access
- **Implementation**: Private constructor, static instance, static getter
- **Thread-safe implementation**: Use eager initialization or synchronization
- **Example**: Database connection pool, logging

### 331. How to create thread-safe singleton?
- **Eager initialization**: Create instance at class loading
- **Lazy initialization with synchronized**: Synchronize getInstance() method
- **Double-checked locking**: Check before and after acquiring lock
- **Bill Pugh Singleton**: Use static inner class (preferred)

### 332. What is factory design pattern?
- Creates objects without specifying exact classes
- **Benefit**: Decouple creation from client
- **Example**: Connection factory for different databases
- **Variants**: Simple factory, Factory method, Abstract factory

### 333. What is prototype design pattern?
- Create objects by copying existing object (prototype)
- **Java implementation**: Implement Cloneable interface; override clone()
- **Advantage**: Cheaper than creating new object; good for complex objects

### 334. What is observer design pattern?
- One-to-many relationship; object notifies dependents of state changes
- **Components**: Subject (Observable), Observer (Listener)
- **Example**: Event listeners, property change listeners

### 335. What is strategy design pattern?
- Define family of algorithms; make them interchangeable
- **Components**: Strategy interface, concrete strategies, context
- **Example**: Different sorting algorithms, payment methods

### 336. What is decorator design pattern?
- Add responsibilities to object dynamically
- **Advantage**: Alternative to subclassing; more flexible
- **Example**: BufferedInputStream wrapping FileInputStream

### 337. What is adapter design pattern?
- Convert interface of class to another interface clients expect
- **Use case**: Integrate incompatible classes
- **Example**: Adapting legacy code to new interface

### 338. What is facade design pattern?
- Provide unified interface to complex subsystem
- **Benefit**: Simplify usage; hide complexity
- **Example**: Spring's JdbcTemplate wrapping JDBC

### 339. What is proxy design pattern?
- Provide placeholder for another object
- **Use case**: Lazy loading, security, logging
- **Example**: Spring's @Transactional creates proxy

### 340. What is MVC design pattern?
- Separate application into Model, View, Controller
- **Model**: Data and business logic
- **View**: Presentation
- **Controller**: Handles input; updates model

- **Difference**: @Mock for dependencies; @InjectMocks for class being tested

### 344. What is test-driven development (TDD)?
- Write tests before implementation
- **Cycle**: Red (test fails) → Green (implement minimum code) → Refactor
- **Advantage**: Forces thinking about requirements; high coverage

### 345. What is integration testing?
- Test multiple components working together
- **Scope**: Multiple layers (database, service, controller)
- **Framework**: Spring Test, @SpringBootTest
- **vs Unit testing**: Broader scope, slower, more realistic

### 346. What is Postman?
- Tool for testing REST APIs
- **Features**: Send requests, view responses, collection of requests
- **Use case**: Manual API testing, API documentation

### 347. What is Swagger/OpenAPI?
- Framework for documenting REST APIs
- **Benefits**: Auto-generated documentation, interactive testing
- **Springfox**: Spring implementation of Swagger
- **Access**: http://localhost:8080/swagger-ui.html

## Additional Scenario-Based Questions

### 367. How would you optimize a slow API endpoint?
- Analyze bottleneck: Database, logic, network
- **Database optimization**: Add indexes, pagination, caching, query optimization
- **Logic optimization**: Reduce unnecessary processing, parallel processing
- **Caching**: Implement Redis or application cache
- **Monitoring**: Use profiler to identify slow code

### 368. How to handle 1000+ concurrent users?
- **Horizontal scaling**: Load balance across multiple instances
- **Caching**: Reduce database queries
- **Database optimization**: Replication, connection pooling
- **Async processing**: Queue long-running tasks
- **CDN**: Cache static resources
- **Monitoring**: Monitor performance under load

### 369. How to implement distributed transaction across microservices?
- **Challenge**: ACID guarantees across distributed systems difficult
- **Solutions**:
  - Saga pattern: Choreography (event-driven) or Orchestration (central coordinator)
  - Compensating transactions: Undo operations on failure
  - 2-phase commit: Not recommended (blocking, poor performance)
- **Tools**: Apache Camel, Temporal, Cadence

### 370. How to implement search across multiple indices in Elasticsearch?
- Search across indices: `GET /index1,index2/_search`
- Union results: Combine responses from multiple indices
- Aggregations: Group across indices
- Aliases: Create alias pointing to multiple indices

### 371. How to handle large file uploads?
- Chunked upload: Upload in parts; resume on failure
- Progress tracking: Show upload progress to user
- Temporary storage: Store chunks temporarily; combine on completion
- Compression: Compress before upload
- Async processing: Return immediately; process asynchronously
- Cleanup: Remove incomplete uploads after timeout

### 372. How would you implement real-time notifications?
- **WebSocket**: Two-way communication for real-time push
- **Server-Sent Events (SSE)**: Server-to-client push
- **Long polling**: Client polls server periodically (less efficient)
- **Message queues**: RabbitMQ for asynchronous notifications
- **Pub-Sub pattern**: Clients subscribe to relevant topics

### 373. How to prevent SQL injection?
- **Parameterized queries**: Use PreparedStatement with placeholders
- **ORM frameworks**: Hibernate/JPA handle escaping
- **Input validation**: Validate and sanitize user input
- **Least privilege**: Database user with minimal permissions
- **Example (Wrong)**: `SELECT * FROM users WHERE id = " + id`
- **Example (Right)**: `SELECT * FROM users WHERE id = ?` with bound parameter

### 374. How to implement caching strategy?
- **Cache-aside**: Application checks cache first; load from DB if miss
- **Write-through**: Write to cache and DB together
- **Write-behind**: Write to cache first; asynchronously to DB
- **Expiration**: TTL (Time To Live) for cache entries
- **Invalidation**: Proactive invalidation on data change
- **Tools**: Redis, Memcached, Spring Cache
